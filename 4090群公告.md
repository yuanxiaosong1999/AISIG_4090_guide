## 🔥 一、磁盘问题

`/data` 和 `/newdata2` 已经 **100% 写满**。

1. **不要再往 `/data`、`/newdata2` 写新的大文件、模型、数据集**
2. **需要长期保存的数据/模型，统一放 `/newdata/home/<你的名字>/`**
   - 自己建一个以自己用户名命名的文件夹：`mkdir -p /newdata/home/<你的名字>`
   - 没权限建？联系管理员
3. **HuggingFace 缓存改到 /newdata**：
   ```bash
   echo 'export HF_HOME=/newdata/home/<你的名字>/hf_cache' >> ~/.bashrc
   source ~/.bashrc
   ```
4. **临时文件**可以继续在原位，用完请清理

## 🖥️ 三、连接方式

```bash
ssh -p 23222 <你的用户名>@<服务器IP>
```

⚠️ **端口是 23222，不是默认 22**。网上教程默认都是 22，记得自己加 `-p 23222`。

校外访问需要先连**学校 VPN**。

## 🎮 四、GPU 使用

1. **跑任务前先 `nvtop` 看哪张卡空闲**
2. **用 `CUDA_VISIBLE_DEVICES` 明确指定**，别默认占所有：
   ```bash
   CUDA_VISIBLE_DEVICES=2 python train.py
   ```
3. **训练完 kill 进程**，别让 python 挂着空占显存
4. **大任务（>24h 或占 4+ 张卡）提前群里报备**
5. **不要直接 kill 别人的进程**，先群里确认

每张 4090 是 **48GB** 显存（不是公版 24GB），算 batch size 时按 48GB 算。

⚠️ **自动清理提醒**：服务器上有监控脚本（已运行一段时间），**GPU 利用率持续 0% 的进程会被自动 kill**——白天 2 小时、夜间（22:00-08:00）4 小时。详见文档第七章，重点看"哪些情况容易被误杀"和"怎么避免"。

## 🌐 五、访问外网（GitHub / HuggingFace）

推荐用 **SSH 端口转发**（把你本地的代理借给服务器）：

```bash
# 连服务器时加 -R 参数（本地有代理在 7890）
ssh -p 23222 -R 7890:127.0.0.1:7890 <你的用户名>@<服务器IP>

# 登录后启用代理
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
```

⚠️ **不要把 `export http_proxy=` 写进 `~/.bashrc`**，没开转发时所有网络命令会全报错。

### VSCode Remote-SSH 怎么配置代理转发

在你**本地电脑**的 `~/.ssh/config` 加一段（这一份配置同时给命令行 ssh 和 VSCode 用）：

```
Host lab4090
    HostName <服务器IP>
    Port 23222
    User <你的用户名>
    RemoteForward 7890 127.0.0.1:7890
```

VSCode 里 `Remote-SSH: Connect to Host...` → 选 `lab4090`，自动带代理转发。
之后服务器上的终端只要 `export http_proxy=...`（同上）就能用代理。

服务器上 `/opt/clash` 的代理来源不明、不建议依赖。

## ⚠️ 六、Docker 权限

**99% 的训练/调试用 conda 就够了，不需要 Docker**（见详细文档第四章）。

Docker 权限**默认不开**。docker 组成员技术上等同 root：能读取所有人的私有文件、密码哈希、SSH 密钥，且**操作不会留 sudo 日志**。

确实需要的场景（部署长期服务、跑别人发布的镜像、依赖系统级软件 conda 装不了），找管理员说明用途和挂载目录后开通。

**已经开了 docker 权限的同学请严格遵守**：

- ✅ **只挂载自己的目录**：`-v /newdata/home/<你>:/workspace`
- ❌ **不要挂载别人的目录、`/`、`/etc`、`/home`、`/data`**
- ❌ **不要用 `--privileged`、`--pid=host`、`--net=host`** 这类危险标志
- ⚠️ **删除别人的文件不会有系统拦截**（docker 内是 root），**造成的麻烦个人负责**

## 📋 七、其他必须知道的规矩

- **不要动别人的目录**，技术上能进也别进
- **不要修改系统配置**（`/etc/`、`/opt/`、systemd 等），有需要找管理员
- **长任务用 `tmux`**，断网不中断
- **密码不要用弱密码**
